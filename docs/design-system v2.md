## Anti-Convergence Rules (Read First)

Claude Code tends toward generic outputs. The following are **explicitly banned** from every file in this project:

### Banned Fonts
- Fraunces (current site, Claude default)
- DM Sans (current site, Claude default)
- Inter (most common AI default)
- Roboto (second most common AI default)
- Space Grotesk (Claude convergence pattern)
- Arial, system-ui, sans-serif as primary

### Banned Colors
- #080808, #0f0f0f, #111111, #141414 as backgrounds (AI dark-mode default)
- #c8965a or any amber/gold as accent (current site, AI default)
- Purple gradients on any background
- Single-accent-on-dark-background patterns

### Banned Effects
- SVG noise/grain overlays at any opacity
- Fade-in translateY scroll reveal as primary animation
- Film grain, static, or noise textures
- Gradient mesh backgrounds

### Banned Patterns
- 900px max-width centered column for all content
- Uniform vertical section rhythm (same padding/gap everywhere)
- Dark cards (#141414) with colored left border
- Identical section padding throughout the page

**If any element in a build looks like it could have been generated by an AI coding tool, refactor it until it doesn't.**

## Aesthetic Direction

**Warm editorial.** Think: Pentagram portfolio meets McKinsey Digital meets the Financial Times. The site should feel like it was designed by someone who reads widely and thinks carefully — not by someone who prompted a coding tool. Authoritative without being corporate. Warm without being casual. Light without being sterile.

The design north star is **typographic authorship** — every spacing decision, every weight choice, every color application should feel deliberate and authored. The typography never asks for attention. It assumes it.

### The Stanford Constraint
The Stanford Web Credibility Project found that sites deemed "too slick" lost credibility. This design system threads the needle: craft over polish, warmth over perfection, editorial restraint over visual pyrotechnics.

## Typography

### Fonts
- **Display / Headings:** Newsreader (Google Fonts) — variable, optical size
  - Hero: weight 300 (light, sharp)
  - Section titles: weight 400
  - Emphasis: weight 500 italic
  - Pull quotes: weight 300 italic
- **Body / UI:** General Sans (Fontshare, self-hosted WOFF2)
  - Body text: weight 400 (Regular)
  - Labels/nav: weight 500 (Medium)
  - Buttons/emphasis: weight 600 (Semibold)
  - Light captions: weight 300 (Light)

### Type Rules
- Headlines use Newsreader. Everything else uses General Sans.
- Never combine more than 2 weights of Newsreader on a single screen.
- Body text is always #4a4a5e (--text-secondary), not --text-primary. Reserve --text-primary for headings.
- No bold body text. Use --text-primary color or General Sans 500 for inline emphasis.
- Pull quotes use Newsreader 300 italic at 1.3rem.

## Animations

### Philosophy
Motion serves comprehension, not decoration. Every animation must answer: "What does this help the user understand?"

### Allowed Animations
- **Page load:** Staggered opacity reveal (0 → 1, no translateY) with 100ms delays between elements. Hero content appears first, then secondary content.
- **Hover states:** Color transitions (0.2s ease), subtle shadow elevation on cards (0.2s), button scale on press (0.98, 0.1s).
- **Assessment flow:** Immediate transitions. No delays between question selection and next question. Results bars fill with 0.8s ease.
- **Scroll-triggered:** Allowed ONLY for the framework page layer transitions and homepage section reveals. Use opacity only (0 → 1), threshold 0.15. No translateY, no translateX, no scale.

### Banned Animations
- translateY scroll reveals (AI default)
- Parallax effects
- Bouncing or elastic easing
- Decorative loading spinners
- Any animation longer than 1s

### Reduced Motion
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## Regression Prevention Rules

**These rules are non-negotiable. Every Claude Code prompt for R1 must comply.**

### The Safe Path: Values-Only Swap
The entire redesign operates on one principle: **keep all CSS variable NAMES and all CSS class NAMES identical. Change only their VALUES.**

- `--accent` stays `--accent` → value changes from `#c8965a` to `#722F37` (claret)
- `--font-display` stays `--font-display` → value changes from `'Fraunces', serif` to `'Newsreader', Georgia, serif`
- `.option-button` stays `.option-button` → styles change, class name preserved
- `.verdict-badge` stays `.verdict-badge` → colors change via CSS variables

This means: **zero JS modifications to assessment-engine.js or assessment-ui.js.** Zero Playwright selector changes. Zero DOM structure changes in the assessment flow.

### Files That Must NOT Be Modified
- `js/assessment-engine.js` — scoring logic, N/A handling, relevance weights, back button state
- `js/assessment-ui.js` — DOM creation, event listeners, state management (CSS variables it reads will automatically resolve to new values)
- `functions/api/v1/*.js` — API routes
- `functions/lib/ai-proxy.js` — Anthropic API calls
- `functions/lib/brief-prompts.js` — brief generation prompts
- `functions/lib/middleware/*.js` — auth, CORS, rate limiting, responses
- `workers/scheduled-brief/index.js` — brief scheduling logic (only email template colors change)
- `tests/e2e/specs/*.js` — test selectors (should not need modification if class names are preserved)
- All D1 migration files
- `wrangler.toml`

### Files That WILL Be Modified
1. `css/style.css` — All CSS custom properties, component styles, section styles, animations
2. `index.html` (homepage) — Font link href, any hardcoded font-family, layout width, section structure
3. `assessment/index.html` — Font link href, hardcoded font-family in inline styles, autofill overrides
4. `framework/index.html` — Font link href, hardcoded font-family
5. `about/index.html` — Font link href, hardcoded font-family
6. `functions/lib/email-templates.js` — All hardcoded #c8965a → #722F37, heading font-family
7. `functions/lib/handlers/email.js` — Dark-mode results email template → light-mode or deprecate
8. `workers/daily-digest-worker/index.js` — Header accent color
9. OG image source files — regeneration with new palette
10. `favicon.svg` — regeneration with new palette
11. New: `/fonts/GeneralSans-Variable.woff2` — added asset
